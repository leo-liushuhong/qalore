# 综合层

当文本适配器和代码适配器均已完成时加载。

两个视角地位对等：文本描述「系统应该做什么」，代码描述「系统实际做什么」。综合层让两个视角互相照亮——代码发现但 PRD 没写的内容，与 PRD 写了但代码没实现的内容，重要程度对等，都是第一公民。

---

## 核心思考哲学：双向交叉验证

遍历两份断言集合，识别同一场景并分类。判断两条断言是否描述同一场景，满足以下任一即是：
1. 触发条件相同 + 受影响对象属同一业务实体
2. 两条断言描述同一个可观察的系统状态变化
3. 维度标签相同，操作对象在业务语义上等价（「用户」≈ `user_id`，「标题字段」≈ `title param`）

**分类处理：**

```
两者描述同一场景
→ 合并为一条更丰富的断言
  · 用户侧可观察表述作为主断言
  · 代码实现细节作为 ↳ 补充行（有额外信息时追加，无则省略）
  · assert ID 对齐：代码断言 ID 改写为与业务断言相同的值
  · 置信度写回：业务逻辑.md 对应断言行追加 `（✓）`

仅文本有，代码无
→ 独立保留，业务逻辑.md 追加 `（待代码确认）`
→ 提示用户：PRD 描述了这个行为，代码中未找到对应实现

仅代码有，文本无
→ 独立保留，视为完整的独立断言，与文本断言同等地位
→ 业务逻辑.md 末尾追加「代码独立断言」节，标注 `（代码独立）`

两者描述同一场景但结论不同 ⚠️
→ 不合并，两个版本均保留
→ 业务逻辑.md 追加 `⚠️` 标注及两个版本描述
→ 明确列出冲突，建议与 PM/开发确认
```

无法确定是否同一场景时，保持独立，不强行合并。

## 产出

**多源场景下的唯一【待沉淀】出口。** 两个适配器均不单独输出【待沉淀】，由本层统一汇总后输出一张确认表。

**pending_count 合并规则：** 将文本适配器和代码适配器各自产生的 pending 项数量相加，得到最终的 `status.pending_count` 写入 index.json 声明。两者分别产出的 pending 项各自保留在对应文件（业务逻辑.md 和代码逻辑.md），不合并文本。

输出统一交接块，格式定义见 `{practices_path}/tech-stacks/functional/story-formats.md`「统一交接块格式」章节：

```
【测试意图已提炼】
项目：{项目名}
模块：{模块名}
信息源：混合（文本 + 代码）

可测试断言：
  [assert-IMG-001] 【功能】（✓）提交按钮 · 表单完整时点击 → 订单创建成功
    ↳ 实现侧：submit() 写入 DB，发布 order.created 事件
  [assert-IMG-003] 【输入约束】（待代码确认）标题字段 · 超过 50 字符 → 显示校验提示
  [assert-IMG-005] 【边界值】（代码独立）config.retry · 值为 0 → 不重试，直接返回失败
  [assert-IMG-004] 【状态转换】⚠️ 审核状态取消行为
    文本：审核中状态可被取消
    代码：cancel() 对 reviewing 抛出 InvalidStateError

待确认项（有则输出，无则省略）：
  【{维度}】{对象}：{具体不明确内容}

回归候选（无则省略）：
  · {模块名}：code_paths 与本次分析路径重叠，建议验证其受影响功能点
```

冲突摘要（若有⚠️项，在【待沉淀】上方先行输出）：
```
【冲突提示】以下场景在文本与代码之间存在矛盾，建议与 PM/开发确认后再生成用例：
  · {对象}：文本描述「{A}」，代码实现「{B}」
```

汇总两个适配器的写入声明，输出一张合并的【待沉淀】：

```
【待沉淀】
| 文件 | 路径 | 操作 | 变更摘要 |
|------|------|------|---------|
| {模块名}-功能-业务逻辑.md | {story_path}/{项目}/{模块}/ | {新建/patch} | 新增 {n} 条断言，现有 {m1} 条 → 变更后 {m1+n} 条 |
| {模块名}-功能-代码逻辑.md | {story_path}/{项目}/{模块}/ | {新建/patch} | 新增 {k} 条断言，现有 {m2} 条 → 变更后 {m2+k} 条 |
| index.json | {story_path}/{项目}/ | {新建条目/patch} | assert_seq={最终值}，pending_count={文本pending数+代码pending数}，{其他变更字段} |

新增断言预览（供用户确认内容正确性）：
{按文件分组，逐行列出全部新增断言}
```
