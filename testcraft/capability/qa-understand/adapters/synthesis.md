# 综合层

当文本适配器和代码适配器**均已完成**时加载，执行双重验证、合并断言、写回置信度标注，产出统一交接块。
两个适配器的断言集合是**平行独立的视角**，综合层负责：识别同一场景并合并、标注置信度、将结果持久化写回业务逻辑.md。

---

## 触发条件

context 中同时存在：
- 文本适配器的断言集合（来自 7问追问）
- 代码适配器的断言集合（来自五镜头追问）

单源场景不加载本文件，由对应适配器直接输出交接块。

---

## 双重验证逻辑

对每条文本断言，在代码断言中查找对应的实现覆盖：

```
一致（文本有，代码也有，描述的是同一个系统行为）
→ 合并为一条更丰富的断言：
    · 用户侧可观察表述（来自文本视角）作为主断言
    · 代码层面的实现细节（来自代码视角）作为 ↳ 补充行（有额外技术细节时追加）
    · assert ID 对齐：将代码断言的 ID 改写为与业务断言相同的值
    · 置信度写回：在业务逻辑.md 对应断言行追加 `（✓）` 标注（持久化，跨会话可读）

  交接块格式：
  [assert-IMG-001] 【维度】（✓）对象 · 条件 → 用户侧可观察预期
    ↳ 实现侧：{代码层面的具体行为}（可选，无额外信息时省略此行）

仅文本有，代码无
→ 保持独立，在业务逻辑.md 对应断言行追加 `（待代码确认）` 标注（持久化）
→ 向用户提示：PRD 中有描述，代码中未找到对应实现，
  建议与开发确认是漏实现还是 PRD 超前

仅代码有，文本无
→ 在业务逻辑.md 末尾追加「代码补充断言」节，写入该断言并标注 `（仅实现视角）`（持久化）
→ 不预判是否应该测试，由 qa-functional-test 按覆盖策略决定

文本与代码存在冲突（描述同一场景但结论不同）⚠️
→ 不合并，在业务逻辑.md 对应断言行追加 `⚠️` 标注及两个版本描述（持久化）
→ 向用户明确列出冲突，建议与 PM/开发确认以哪个为准
```

**同一场景的识别准则（按优先级逐条判断，满足任一即视为同一场景）：**

1. 触发条件相同 + 受影响对象属同一业务实体 → 同一场景
2. 文本断言的「可观察预期」与代码断言的「实现结果」描述同一个系统状态变化 → 同一场景
3. 两条断言的维度标签相同，且操作对象在业务语义上等价（如「用户」≈ `user_id`，「标题字段」≈ `title param`）→ 同一场景

**不合并的情况：**
- 触发条件相同但预期结果不同 → 冲突，标 ⚠️ 需确认，不合并
- 仅维度相同但对象不同 → 独立断言，不合并
- 无法确定是否同一场景时 → 保持独立，不强行合并

## 统一断言集合产出

综合层完成验证后，将所有断言合并为一份统一断言集合，
按 qa-understand/SKILL.md 定义的交接块格式输出：

```
【测试意图已提炼】
项目：{项目名}
模块：{模块名}
信息源：混合（文本 + 代码）

可测试断言：
  【功能】（✓）提交按钮 · 表单完整时点击 → 订单创建成功，页面跳转至详情页
    ↳ 实现侧：submit() 写入 DB，发布 order.created 事件
  【权限】（✓）未登录用户 · 访问付费内容 → 跳转至登录页
  【输入约束】（待代码确认）标题字段 · 超过 50 字符 → 显示校验提示
  【边界值】（仅实现视角）config.retry · 值为 0 → 不重试，直接返回失败
  【状态转换】⚠️ 需确认
    文本：审核中状态可被取消
    代码：cancel() 对 reviewing 状态抛出 InvalidStateError

待确认项：
  【{维度}】{对象}：{具体不明确内容}

回归候选（code_paths 交集检测，无则省略此节）：
  · {模块名}：code_paths 与本次分析路径重叠，建议验证其受影响功能点
```

## 产出写入声明

综合层汇总两个适配器的写入声明，合并输出**一张**【待沉淀】声明。
**text.md 和 code.md 的【待沉淀】节在多源场景下不单独输出，由本层统一输出。**

冲突摘要（若有）在声明上方输出后，再展示合并后的【待沉淀】表。

**多字段合并规则：**
- `status.pending_count`：取两个适配器 pending 数量之和
- `assert_seq`：取两个适配器分配的最大序号（synthesis 对齐后可能调整，以最终值为准）
- `code_paths`：来自代码适配器，文本适配器不写此字段，无需合并
- `status.business_logic` / `status.code_logic`：各自独立，无冲突
- `last_updated`：取当前日期，覆盖两者
